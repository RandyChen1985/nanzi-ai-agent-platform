# 数据门户首页重组设计

## 目标

将当前以场景卡片和数据目录为主的抽屉式“数据门户导航”，重组为面向日常使用的“我的数据首页”。首页优先回答“今天需要关注什么、最近分析到哪里、常用报表在哪里”，同时保留现有推荐场景、快捷提问、数据字典和抽屉入口。

本阶段不替换 ChatBI，不重写黄金报表执行链路，也不引入尚未确认的指标告警规则。所有首页状态必须来自现有运行历史、订阅、通知、报表和数据门户接口，不允许使用模型补造运行状态。

## 产品形态

### 完整页面

新增 `/dashboard/data-portal` 页面，桌面端使用左侧二级导航，移动端使用底部导航。页面包含：

1. 数据首页。
2. 我的报表。
3. 推荐场景。
4. 数据目录。

页面沿用 Dashboard 布局和现有 `menu:ai_chat` 权限，不在首期新增菜单权限或数据库迁移。入口包括 Dashboard 导航、聊天数据门户抽屉中的“打开完整页面”，以及报表通知跳转。

### 快捷抽屉

保留 `DatasetPortalDrawer.vue`，继续用于聊天中的快捷查找、快捷提问和钉住浏览。抽屉不复制完整首页，只增加“打开完整页面”入口。抽屉和完整页面复用相同 composable、黄金报表组件和场景目录组件。

## 信息架构

### 数据首页

#### 今日关注

首期只展示有真实来源的三项：

- 运行失败：当前用户有权访问的黄金报表在今日产生的失败运行数，并展示最近一条失败记录。
- 今日新简报：今日成功生成的订阅智能简报数，并展示最近生成时间。
- 订阅运行状态：当前用户启用的订阅数和今日已完成数。

用户已经要求暂不实现告警，因此不展示“异常指标”卡片，不预留假数据占位。某项为零时显示正常状态；接口失败时该项显示“暂时无法获取”，其他区域继续可用。

#### 最近分析

统一展示最多 6 条最近活动：

- 最近黄金报表运行。
- 最近订阅智能简报。
- 最近 ChatBI 数据会话。

每条活动必须标明来源、时间和可用动作：查看简报、打开运行详情、继续分析。ChatBI 会话若没有可复用的结构化结果，只允许打开会话，不显示“继续分析”。

#### 我的报表

首页展示最多 6 张报表，提供以下筛选：

- 已订阅。
- 置顶。
- 收藏。
- 共享给我。
- 最近运行。

报表卡片展示标题、所有者或执行智能体、订阅频率、最近运行状态和时间。点击卡片打开现有报表详情；首页不复制编辑、订阅和运行历史表单。

#### 推荐场景与数据目录

推荐场景继续使用现有 `dataset-menu` 返回的场景分组、快捷问题和“换一批”能力，但放在首页下半区。数据目录首屏只显示数据集、数据表和字段数量及入口，详细表和字段字典进入“数据目录”模块查看。

### 我的报表完整模块

复用现有黄金报表浏览、筛选、运行、详情、历史和订阅能力。完整模块使用较宽列表或卡片布局，不继续受 28rem 抽屉宽度限制。筛选状态通过 query 参数保存，例如 `?section=reports&filter=subscribed`，便于通知和任务中心直接定位。

### 推荐场景完整模块

复用 `DatasetCapabilityMenu.vue` 中的场景与推荐问题能力，但将黄金报表区从该组件拆出，避免完整页面再次出现“一个大组件包办所有区域”。场景提问继续支持直接发送与填入输入框；从完整页面发起提问时跳转 ChatBI 并携带问题，不在门户页面内嵌第二套聊天窗口。

### 数据目录完整模块

复用现有授权数据集目录和字段字典。首期支持搜索数据集、表和字段，以及从表或字段发起结构说明、查询明细和推荐提问。指标目录留到第四阶段指标语义层，不在本期用字段信息模拟指标。

## 前端结构

新增：

- `frontend/src/views/DataPortalHome.vue`：完整页面容器与路由状态。
- `frontend/src/components/data-portal/DataPortalOverview.vue`：今日关注和最近分析。
- `frontend/src/components/data-portal/DataPortalReportSection.vue`：首页报表摘要及筛选。
- `frontend/src/components/data-portal/DataPortalSceneSection.vue`：推荐场景适配层。
- `frontend/src/components/data-portal/DataPortalCatalogSection.vue`：数据目录摘要和完整目录。
- `frontend/src/composables/useDataPortalHome.ts`：首页聚合数据加载、刷新和局部错误状态。

调整：

- `DatasetCapabilityMenu.vue` 保留兼容入口，但逐步把黄金报表区和场景区抽成可复用子组件。
- `DatasetPortalDrawer.vue` 增加打开完整页面按钮。
- `useDatasetPortal.ts` 继续负责抽屉、场景缓存与快捷提问，不承担首页运行摘要。

正式聊天页和调试页继续使用同一抽屉组件，不在两个页面分别实现首页逻辑。

## 后端接口

新增 `GET /api/v1/chat/data-portal/home` 聚合接口，避免首页并发调用大量明细接口。返回稳定结构：

```json
{
  "attention": {
    "failed_runs_today": 0,
    "latest_failed_run": null,
    "digests_today": 0,
    "latest_digest_at": null,
    "active_subscriptions": 0,
    "completed_subscriptions_today": 0
  },
  "recent_analysis": [],
  "report_summary": {
    "subscribed": 0,
    "pinned": 0,
    "favorite": 0,
    "shared": 0,
    "recent": 0,
    "items": []
  },
  "generated_at": "2026-07-15T09:00:00+08:00"
}
```

接口只聚合当前用户可访问的数据：

- 报表和分享权限复用黄金报表现有可见性规则。
- 运行记录复用运行历史权限判断。
- 订阅和通知按当前用户隔离。
- ChatBI 最近会话只返回当前用户拥有的会话，并只暴露首页需要的标题、时间、会话 ID 和是否存在可信结构化结果。

场景和目录继续使用现有 `GET /api/v1/chat/dataset-menu`，避免把 AI 生成场景的慢请求阻塞首页核心数据。

## 数据加载与刷新

页面首次打开时并行请求首页聚合接口和现有数据门户接口：

- 首页聚合接口先完成即可展示今日关注、最近分析和报表。
- 场景接口较慢时显示现有骨架和降级提示，不阻塞上半区。
- 手动刷新同时刷新聚合接口和场景接口；两个结果分别反馈，不因其中一个失败清空另一个已有结果。
- 页面保留上一次成功数据，后台刷新时不闪回空态。

首页聚合结果不做长时间 Redis 缓存，依赖数据库实时查询和短时前端去重。场景目录继续沿用现有缓存、静默恢复和“换一批”去重机制。

## 导航和上下文恢复

- 查看报表：打开现有报表详情，并通过 `report_id`、`run_id` 定位运行记录。
- 查看简报：打开对应运行详情的智能简报内容。
- 继续分析：跳转 `/dashboard/chat`，携带 `report_id`、`run_id` 或 `conversation_id`，复用可信结果快照。
- 推荐问题：跳转 ChatBI 后发送或填入问题，并携带现有数据集/物理表上下文。
- 浏览器前进后退需要恢复门户 section、报表筛选和详情定位。

## 桌面端与移动端

桌面端使用完整页面和左侧二级导航；内容区最大化利用宽度，报表卡片为两到三列。移动端使用底部四项导航，首页按“今日关注、最近分析、我的报表、推荐场景、数据目录”纵向排列，报表改为单列。

移动端不使用横向悬浮大表格；最近分析只保留标题、时间、状态和一个主动作。智能简报继续沿用当前移动端优先的短段落排版。

## 空态与异常

- 无订阅：展示“还没有订阅报表”和“查看黄金报表”入口，不展示全零统计墙。
- 无最近分析：展示一个轻量空态和“开始 ChatBI 查询”入口。
- 无数据集权限：保留现有无权限说明，不影响用户查看已经共享且仍有执行权限的报表。
- 聚合接口局部数据异常：通过字段级 `unavailable` 状态展示，不让整个首页失败。
- 场景模型失败：继续展示确定性基础目录，并沿用现有失败说明。

## 首期范围与后续范围

首期实现：

1. 完整页面路由和桌面/移动端信息架构。
2. 今日关注中的失败运行、智能简报和订阅状态。
3. 最近黄金报表运行、智能简报和可识别的最近 ChatBI 数据会话。
4. 我的报表摘要和现有筛选复用。
5. 推荐场景与数据目录下沉复用。
6. 抽屉打开完整页面入口。

首期不实现：

- 阈值或异常指标告警。
- 指标中心和指标口径治理。
- 门户内嵌完整 ChatBI。
- 新的报表分享、多人订阅或评论体系。
- 预测、自动根因分析和运营驾驶舱。

## 验收标准

- 用户打开完整页面后，不等待场景模型即可看到真实的报表和运行信息。
- 首屏优先展示今日关注、最近分析和我的报表，推荐场景与目录位于下半区。
- 所有统计、状态和时间均能追溯到当前用户有权访问的数据库记录。
- 报表、简报和继续分析跳转能够恢复对应 `report_id`、`run_id` 或会话上下文。
- 抽屉原有快捷提问、钉住、刷新、黄金报表和移动端行为不回归。
- 桌面端 1280px 及移动端 375px 下无横向溢出，主要操作不重叠。
- 首页聚合接口、权限隔离、空态、局部失败和前端路由状态均有自动化测试。
