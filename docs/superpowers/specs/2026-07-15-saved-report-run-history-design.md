# 黄金报表运行历史设计

## 目标

把黄金报表从“可重复运行的 SQL”推进到可追溯的报表资产：每次手动运行都记录状态、参数、耗时、行数、权限提示、失败原因及有限结果快照，并在报表详情中可查看。

## 范围

- 新增 `portal_saved_report_runs` 表。
- 黄金报表手动执行成功或失败时写入运行记录。
- 新增运行历史列表与单次运行详情接口。
- 报表详情抽屉增加“报表信息 / 运行历史”页签。
- 结果快照最多保存前 200 行；总行数单独记录。

本阶段不包含定时订阅、阈值告警、消息通知和历史数据导出。

## 数据与权限

- 所有者可查看该报表全部运行记录。
- 共享使用者仅可查看自己触发的记录。
- 列表返回摘要；执行 SQL 与结果快照仅由详情接口返回。
- 行权限提示只保存必要字段，不保存额外用户上下文。
- 运行参数与 SQL 仍属于敏感信息，接口访问始终复用报表本身的访问校验。

## 失败语义

- SQL 校验、表权限和执行异常均写入 `error` 记录。
- 参数在 SQL 解析前即不合法时，不产生运行记录，因为尚未形成一次可执行运行。
- 失败记录与报表当前状态在抛出 HTTP 错误前显式提交，避免请求回滚丢失历史。

## 前端体验

- 历史列表展示运行时间、状态、耗时、数据行数和触发人范围。
- 点击记录后按需加载详情，展示参数、SQL、权限提示、失败原因和前 200 行快照。
- 空状态、加载态和失败态均在抽屉内完成，不打断报表浏览。
