# ChatBI 可信说明与继续分析设计

## 目标

在不改变 ChatBI 现有 SQL、门禁、修复和回答正文的前提下，为成功查数消息增加结构化“数据依据”，并将固定“可视化分析”升级为单一“继续分析”入口。桌面端使用 Popover，移动端使用底部弹层展示动态推荐动作。

## 数据契约

后端在成功查数后发送 `chatbi_insight_meta` SSE 事件。事件包含版本、数据来源、权限提示、执行模式、结果结构、最终 SQL 和推荐动作。旧客户端可安全忽略该事件；新客户端缺少事件时保持现有行为。

数据依据只能从真实运行态、Schema 绑定、SQL 工具参数和权限执行结果提取，禁止让模型生成数据来源、权限状态或执行状态。权限表达式、连接信息和内部推理不得进入前端契约。

## 继续分析

推荐动作由确定性规则根据结果列结构生成，最多四项。时间列与数值列推荐趋势；分类列与数值列推荐排名和贡献度；明细结果推荐异常分析；无法可靠识别时退回可视化分析、总结关键结论和导出数据。

消息区只展示一个“继续分析”按钮。桌面端点击后打开邻近 Popover；移动端打开底部 Action Sheet。动作可直接发送追问或填入输入框。

## 兼容与安全

- 不修改现有回答正文和 SQL 执行结果。
- 不改变 Schema、SQL、权限、沙箱、修复和联邦查询门禁。
- 最终 SQL 优先使用权限改写后的实际执行 SQL。
- 没有结构化元数据时继续展示旧的“可视化分析”能力。
- 正式聊天页和调试页复用相同类型、组件和事件归并函数。

## 验收

- 单表、权限过滤、修复后成功场景的数据依据准确。
- 时间汇总、分类汇总和明细结果生成匹配的继续分析动作。
- 无成功结构化结果时不展示依据或错误动作。
- SSE 恢复和重复事件不会造成动作重复。
- 桌面端 Popover 与移动端底部弹层均可操作。
